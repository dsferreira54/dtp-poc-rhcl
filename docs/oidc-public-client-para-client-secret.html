Como mudamos o login da app de “sem senha” para “com senha”

Comparação lado a lado: antes à esquerda, depois à direita. Cada coluna mostra o objeto YAML completo, com as linhas alteradas destacadas em cor.

Como ler os destaques Verde = linha ou bloco novo Amarelo = linha que mudou de valor Vermelho = objeto removido
📋

Resumo em 30 segundos

A app hello-world-app logava no Keycloak só com o nome da aplicação, sem senha secreta. Queríamos exigir também uma senha (client secret). Não bastou mudar o Keycloak — a ferramenta automática de login (OIDCPolicy) não sabe enviar senha. Trocamos por 4 arquivos de configuração manual. A app Hello World em si não mudou.

🔓

Antes

1 arquivo automático (OIDCPolicy). Keycloak sem senha. Qualquer endereço de retorno aceito.

🔐

Depois

3 regras escritas à mão + 2 cofres de senha. Keycloak exige senha. Endereço de retorno fixo.

Antes — public client

  • Só o nome da app
  • 1 YAML automático
  • Sem senha no Keycloak

Depois — com client secret

  • Nome + senha secreta
  • 4 arquivos configurados
  • Senha guardada no cluster
💡
Analogia Antes era como entrar no prédio só dizendo “sou o João”. Depois precisa do nome e de um crachá com senha — que só o porteiro (gateway) guarda, nunca o visitante (browser).
Por quê

Por que não deu só para mudar o Keycloak?

Marcamos no Keycloak que a app agora precisa de senha (publicClient: false). Só que quem completa o login no gateway ainda usava a config antiga — sem enviar senha. E essa config antiga (OIDCPolicy) não tem campo para senha; se corrigíssemos na mão, o sistema desfazia sozinho. Por isso apagamos o atalho automático e escrevemos as regras manualmente.

Passo a passo

Ordem para fazer a mudança

  1. Colocar nome e senha no values.yaml
  2. Criar o arquivo do cofre do gateway
  3. Atualizar o Keycloak (3-realm-import.yaml)
  4. Apagar OIDCPolicy e colar os 3 objetos novos
  5. Aplicar no cluster
  6. Se Keycloak já existia: ajustar client na mão
  7. Testar login demo / demo123 no browser

Termos

Glossário

Public client
Login só com nome, sem senha secreta
Client secret
Senha da aplicação perante o Keycloak
OIDCPolicy
Atalho automático de login (não suporta senha na v1.4.1)
AuthPolicy
Regra de login escrita manualmente
Secret
Cofre de senhas no Kubernetes
